MinIO test setup and required information

This file explains how to run a local MinIO for testing and what information you must provide so I can run end-to-end tests and upload chapters.

Quick start (Docker):

1. Run MinIO server (example):

```bash
docker run -p 9000:9000 --name minio -e MINIO_ROOT_USER=minioadmin -e MINIO_ROOT_PASSWORD=minioadmin -v ${PWD}/minio-data:/data -d minio/minio server /data
```

2. Install `mc` (MinIO client) or use web UI at http://localhost:9000 with the above credentials.

3. Create bucket and set public-read policy (example using `mc`):

```bash
mc alias set local http://localhost:9000 minioadmin minioadmin
mc mb local/novel-chapters-test
# make public read (for non-locked chapters)
mc policy set public local/novel-chapters-test
```

What you must provide (for me to run end-to-end):

- MINIO endpoint URL (e.g., `http://localhost:9000` or S3-compatible endpoint)
- Access key and secret key for the MinIO instance (or test credentials)
- Bucket name to use (e.g., `novel-chapters-test`)
- Whether SSL is used (`MINIO_USE_SSL=true|false`) — usually `false` for local
- Public URL base (optional) if you want object URLs to use a specific domain (e.g., `https://r2.example.com`)

Environment variables I will read (you can put these in `backend/.env`):

- `MINIO_ENDPOINT` — e.g., `http://localhost:9000`
- `MINIO_ACCESS_KEY` — e.g., `minioadmin`
- `MINIO_SECRET_KEY` — e.g., `minioadmin`
- `MINIO_BUCKET` — e.g., `novel-chapters-test`
- `MINIO_REGION` — e.g., `us-east-1`
- `MINIO_FORCE_PATH_STYLE` — `true` (required for MinIO)
- `MINIO_USE_SSL` — `false` for local
- `MINIO_PUBLIC_URL` — optional public base URL

How I'll use MinIO in the project

- For development/testing I will configure the R2 adapter to fall back to MinIO when `MINIO_*` env vars present.
- For public (non-locked) chapters we'll rely on public-read ACL on the bucket and generate direct URLs using `publicUrlBase` + `/bucket/object`.
- For locked chapters I will use presigned URLs generated by the SDK so the backend can first validate purchase/unlock and then return a short-lived access URL.

Testing uploads manually

You can test uploads with `curl` (replace placeholders):

```bash

```
